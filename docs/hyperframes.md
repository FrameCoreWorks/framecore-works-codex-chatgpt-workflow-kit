# HyperFrames Workflow

HyperFrames is treated as a coded-video workflow path, not as a paid media-provider integration.

Use it when the deliverable is an HTML or code-driven video composition, motion graphic, title sequence, captioned clip, or rendered product video.

Expected route:

1. Motion direction.
2. Storyboard or scene structure.
3. Copy and caption planning.
4. HyperFrames production brief.
5. Render QA.
6. Delivery manifest.

The workflow should define scene timing, animation intent, GSAP motion notes, text timing, visual hierarchy, render checks, and final file expectations.
