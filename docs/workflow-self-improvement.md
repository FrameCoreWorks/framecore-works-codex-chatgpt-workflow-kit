# Workflow Self-Improvement

The workflow self-improvement skill turns completed work into auditable improvement logs and change proposals.

It is explicit-only. It does not run as a hidden daemon, edit workflow files automatically, upload artifacts, run external execution tools, or perform destructive operations.

Every proposal must include:

- problem
- evidence labels
- affected agents, gates, handoffs, skills, or files
- proposed change
- expected benefit
- risk and rollback note
- acceptance test
- decision owner and status

Adoption requires explicit approval.
