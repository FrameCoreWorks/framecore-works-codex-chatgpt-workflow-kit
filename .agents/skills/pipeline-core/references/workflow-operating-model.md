# Workflow Operating Model

Default flow:

1. Intent lock.
2. Workflow route.
3. Brief and reference authority.
4. Evidence when facts, docs, models, costs, or claims may change.
5. Direction and structure.
6. Copy and prompt packs.
7. Tool routing and execution manifest when execution is requested.
8. Asset manifest.
9. QA and iteration.
10. Delivery documentation.
11. Workflow self-improvement only when explicitly requested or enabled as a report-only review.

Parallel work is allowed when dependencies are clear. The workflow-orchestrator remains responsible for state and loopbacks.
